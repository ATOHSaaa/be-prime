## 2. SEOの基本方針

### 検索意図を先に決める

- **メインキーワード**（例: `Prime Day 2026`、`UGREEN 500W 充電器`）
- **検索意図**（Know / Do / Compare）
- **読者が記事読了後に得る答え**

### タイトル・description・初段の役割分担

| 要素 | 役割 | 書き方の例 |
|------|------|------------|
| **title** | 検索結果の見出し。価格・割引を入れてよい | `UGREEN 500W充電器が22,784円（24%OFF）｜単ポート240W・6台同時充電` |
| **description** | メタ用の要約（120〜160字目安）。初段のコピペにしない | `〜が定価29,980円から24%OFFの22,784円になっています。スペックや買い方の見方などについて整理します。` |
| **初段（リード）** | 本文冒頭。誰向けか・いまの状況を具体的に | `〜が定価<strong>29,980円</strong>から<strong>24%OFF</strong>の<strong>22,784円</strong>で販売されています。…` |

- 初段は**結論ファースト**（80〜150字目安）
- **「結論から言うと」は使わない**
- リード末尾に、必要なら商品ページへのリンクを1本入れる

### キーワードの置き方

- **タイトル**: メインキーワードを前半に。32〜40文字目安
- **URL（ファイル名）**: 英小文字・ハイフン

```yaml

  - セール

  - asin: B0XXXXXXXXX

    savings: '24%OFF'      # 任意
    note: 定価29,980円→22,784円（24%OFF）

```

## 4. 記事構成

### 商品セール記事（推奨テンプレート）

1. **リード**（見出しなし）
2. **H2: この商品の要点** — スペック表（`項目 | スペック`）
3. **H2: チェックすべきポイント** — 下記 **H3** で小見出しを分ける
4. **H2: こんな人におすすめ** — 2カラム対比表（`向いている人 | 向いていない人`）
5. **H2: 比較**（任意） — 競合・旧モデルとの比較表
6. **H2: よくある質問** — 質問形式の **H3**
7. **H2: まとめ**

### 特集・攻略記事（Prime Day 等）

`category: deals` を使う。

1. **リード**
2. **H2: 概要・定義**（表で整理）
3. **H2: 攻略の基本** など — 手順は **H3**（番号リストではなく見出し）
4. **H2: カテゴリ別・比較**
5. **H2: よくある質問**
6. **H2: まとめ**

### 見出しルール

| レベル | 使い方 |
|--------|--------|
| **H2** | 大セクション（この商品の要点、チェックすべきポイント、まとめ など） |
| **H3** | H2 配下の小項目（価格の見方、ポート構成、FAQの各質問） |

- **H3 の冒頭に番号は付けない**（`### 1. 価格` ✗ → `### いまの価格と通常価格の差` ○）
- 手順やポイントの列挙は、番号付きリストより **H3 見出し** を優先する
- 見出し名は短く具体的に（`セールでチェックすべきポイント` より `チェックすべきポイント`）

### 書かない・省くもの

- **「セール中の買い方（3ステップ）」のような汎用購入フロー** — 商品カードとリードで足りる。無理に入れない
- **根拠の薄い締めの一文**（「アップグレード候補として検討する価値はあります」等の水増し）
- **クーポンコードの記載** — 変動が大きく、誤情報になりやすい
- **「結論から言うと」**

## 5. 価格・セールの書き方

### 表記

- 価格は **`22,784円`** 形式（**`¥` は本文・frontmatter ともに使わない**）
- 強調は `<strong>22,784円</strong>`（`**22,784円**` でも可。`¥` 始まりは CommonMark で太字にならないため非推奨）
- API が `¥22,784` を返しても、商品カード表示はサイト側で `22,784円` に変換される

### 注意書き（価格を書く箇所では必須）

執筆時点の価格を示したあと、次の趣旨を入れる。

> SNSや他サイトの「安い！」といった声だけを鵜呑みにせず、商品ページの現在価格と参考価格をその都度確認することをおすすめしています。当サイトでもなるべく正確な情報を掲載するように努めますが、情報が古くなっている場合もあります。

- **Amazon.co.jp の表示を正**とする旨は、まとめなどで1回以上書く
- 「〇月〇日まで」等の**期限付き断言は避ける**（または `updatedDate` で更新）

## 6. 表の使い方

| 用途 | 列の例 |
|------|--------|
| スペック | `項目 \| スペック` |
| おすすめ対象 | `向いている人 \| 向いていない人` |
| モデル比較 | `モデル目安 \| 総出力 \| …` |
| 攻略・特集 | `項目 \| 内容`、`カテゴリ \| セール傾向 \| 確認ポイント` |

- **左右が対になる2カラム表**（おすすめ対象など）は、サイト側で左右同幅になる
- 3カラム以上の表は内容量に応じた幅のまま

## 7. 商品カード・アフィリエイト

### 商品カードの自動表示

`products` を frontmatter に書くと、次の位置にカードが出る。

| 位置 | 見出し |
|------|--------|
| 目次の直上 | なし（`showHeading={false}`） |
| 記事末尾 | `購入はこちら（Amazon.co.jp）` |

### 本文への差し込み

価格・購入判断の直後など、読者が行動しそうな箇所に単独行で書く。

```markdown
::product{B0DL9MHFJJ}::
```

ASIN を省略すると、frontmatter の先頭 `products` が使われる。

### リンク・アソシエイト

- **amazon.co.jp へのリンクはすべて自動で `tag=be-prime-22` が付く**（手動で tag を書く必要はない）
- 本文中の `Amazonプライム` / `Prime Day` / `タイムセール` は**各キーワードの初出のみ**自動リンク（rehype）
- `cta: prime-day` などで記事末尾 CTA を切り替え

## 8. 品質チェックリスト

- [ ] タイトルにメインKW・価格/割引（該当する場合）が自然に含まれている
- [ ] description と初段が役割分担されている（同文のコピペになっていない）
- [ ] 初段で結論が述べられ、「結論から言うと」がない
- [ ] 価格は `○○円` 表記で統一されている
- [ ] 価格記載箇所に、公式確認＋当サイトの情報鮮度への注意書きがある
- [ ] H3 に番号が付いていない
- [ ] 不要な「買い方3ステップ」や水増しの締め文がない
- [ ] `products` と `::product{}::` の配置が適切
- [ ] `category` が正しく設定されている
- [ ] セール記事なら `saleEndDate` を検討した
- [ ] `draft: false` で公開
