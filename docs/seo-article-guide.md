## 2. SEOの基本方針

### 検索意図を先に決める

- **メインキーワード**（例: `Prime Day 2026`、`UGREEN 500W 充電器`）
- **検索意図**（Know / Do / Compare）
- **読者が記事読了後に得る答え**

### タイトル・description・初段の役割分担

| 要素 | 役割 | 書き方の例 |
|------|------|------------|
| **title** | 検索結果の見出し。価格・割引を入れてよい | `UGREEN 500W充電器が22,784円（24%OFF）｜単ポート240W・6台同時充電` |
| **description** | メタ用の要約（120〜160字目安）。初段のコピペにしない | `〜が定価29,980円から24%OFFの22,784円になっています。スペックや買い方の見方などについて解説します。` |
| **初段（リード）** | 本文冒頭。誰向けか・いまの状況を具体的に | `〜が定価<strong>29,980円</strong>から<strong>24%OFF</strong>の<strong>22,784円</strong>で販売されています。…` |

- 初段は**結論ファースト**（80〜150字目安）
- **「結論から言うと」は使わない**
- description・リード・まとめで記事の約束を述べるときは、**「〜を整理します」より「〜について解説します」**を優先する
- リード末尾に、必要なら商品ページへのリンクを1本入れる

### キーワードの置き方

- **タイトル**: メインキーワードを前半に。32〜40文字目安
- **URL（ファイル名）**: 英小文字・ハイフン


## 4. 記事構成

### 商品セール記事（推奨テンプレート）

1. **リード**（見出しなし）
2. **H2: この商品の要点** — スペック表（`項目 | スペック`）
3. **H2: チェックすべきポイント** — 下記 **H3** で小見出しを分ける
4. **H2: こんな人におすすめ** — 2カラム対比表（`向いている人 | 向いていない人`）
5. **H2: 比較**（任意） — 競合・旧モデルとの比較表
6. **H2: よくある質問** — 質問形式の **H3**
7. **H2: まとめ**

### 特集・攻略記事（Prime Day 等）

`category: deals` を使う。

1. **リード**
2. **H2: 概要・定義**（表で整理）
3. **H2: 攻略の基本** など — 手順は **H3**（番号リストではなく見出し）
4. **H2: カテゴリ別・比較**
5. **H2: よくある質問**
6. **H2: まとめ**

### 見出しルール

| レベル | 使い方 |
|--------|--------|
| **H2** | 大セクション（この商品の要点、チェックすべきポイント、まとめ など） |
| **H3** | H2 配下の小項目（ポート構成、画質、FAQの各質問） |

- **H3 の冒頭に番号は付けない**（`### 1. 価格` ✗ → `### ポート数と出力のバランス` ○）
- 手順やポイントの列挙は、番号付きリストより **H3 見出し** を優先する
- 見出し名は短く具体的に（`セールでチェックすべきポイント` より `チェックすべきポイント`）

### 書かない・省くもの

- **「いまの価格と通常価格の差」セクション** — リード・タイトル・まとめ・商品カードで価格は伝わる。チェックポイント内で毎回繰り返さない
- **「セール中の買い方（3ステップ）」のような汎用購入フロー** — 商品カードとリードで足りる。無理に入れない
- **根拠の薄い締めの一文**（「アップグレード候補として検討する価値はあります」等の水増し）
- **クーポンコードの記載** — 変動が大きく、誤情報になりやすい
- **「結論から言うと」**

## 5. 価格・セールの書き方

### 表記

- 価格は **`22,784円`** 形式（**`¥` は本文・frontmatter ともに使わない**）
- 強調は `**22,784円**` または `**商品名**` で書く（ビルド時に自動で `<strong>` に変換される）
- `¥` 始まりは CommonMark で太字にならないため非推奨
- 全角括弧 `（` `）`・作品名の `『』`・1文に複数の太字が続く場合でも、`**太字**` で書いてよい
- API が `¥22,784` を返しても、商品カード表示はサイト側で `22,784円` に変換される

### 太字の確認フロー（記事作成・更新時は必須）

記事の Markdown を書き終えたら、**太字が正しくパースされているか**を必ず確認する。

```bash
npm run check:article
```

| コマンド | 内容 |
|----------|------|
| `npm run build` | サイトをビルド |
| `npm run check:bold` | `dist/posts/` 内の記事 HTML に未パースの `**` が残っていないか走査 |
| `npm run check:article` | 上記2つを連続実行（**記事追加・更新時はこちら**） |

- `check:bold` が失敗した場合、本文に `**` がそのまま表示されている箇所がある
- ビルド時に remark プラグインが `**太字**` を `<strong>` へ変換するが、コードブロック内の `**` や記法ミスは検出対象になる
- ローカルプレビュー（`npm run dev`）でも太字表示を目視確認するとよい

### 注意書き

価格の注意書きは**記事全体で1回**で足りる（リード末尾かまとめ）。毎セクションに繰り返さない。

> SNSや他サイトの「安い！」といった声だけを鵜呑みにせず、商品ページの現在価格と参考価格をその都度確認することをおすすめしています。当サイトでもなるべく正確な情報を掲載するように努めますが、情報が古くなっている場合もあります。

- **Amazon.co.jp の表示を正**とする旨は、まとめで1回書く
- 「〇月〇日まで」等の**期限付き断言は避ける**（または `updatedDate` で更新）

## 6. 表の使い方

表には大きく **3種類** ある。種類ごとに Markdown の書き方と見た目が決まる。

### 属性表（キー・バリュー表）

**左に短いラベル、右に説明文**を並べる表。商品スペック、用語の定義、セール概要などで使う。

| 用途 | 列の例 |
|------|--------|
| スペック | `項目 \| スペック` |
| 用語・概要 | `項目 \| 内容` |
| 活用法の整理 | `使い方 \| 回収のイメージ`、`シーン \| Keepaの使い方` など |

**書き方のルール**

- Markdown 上は従来どおり **1行目に `項目 | 内容`（または `項目 | スペック` 等）を書いてよい**。ビルド時に見出し行は自動で除去される
- **画面上では見出し行を出さない**（「項目」「内容」のヘッダーは不要）
- **左列は狭く、右列は広く**表示される（サイト側で自動）
- 左列は短い名詞・ラベル（例: `主な特典`、`型番`、`年会費`）、右列に説明を書く

```markdown
| 項目 | 内容 |
|------|------|
| 主な特典 | 送料無料、Prime Video、Prime Reading など |
| 対象 | Amazonプライム会員 |
```

### 対比表

**左右を対に並べる**表。見出し行は**残す**。

| 用途 | 列の例 |
|------|--------|
| おすすめ対象 | `向いている人 \| 向いていない人` |

- 見出し行 `向いている人 | 向いていない人` はそのまま書く
- **左右同幅**で表示される
- 見出しセルはサイト側で**左を緑系・右をグレー系**に自動着色（向き／不向きの視覚差）

### データ表（比較・一覧）

**3列以上**、または列見出しがそのままデータのカテゴリになる表。

| 用途 | 列の例 |
|------|--------|
| モデル比較 | `モデル目安 \| 総出力 \| …` |
| セール一覧 | `商品 \| セール価格 \| 参考価格` |
| カテゴリ整理 | `カテゴリ \| セール傾向 \| 確認ポイント` |

- **見出し行は残す**（商品名・価格・カテゴリ名などがヘッダーになる）
- 列数に応じた幅のまま表示される

## 7. 商品カード・アフィリエイト

### サムネイル・OGP

`products` を frontmatter に書いた**商品紹介記事**は、ビルド時に商品画像からサムネイルを自動生成する（一覧・トップ・OGP）。**記事ページ冒頭にはサムネイルを表示しない**。

| 優先順位 | 画像ソース |
|----------|-----------|
| 1 | `heroImage`（手動指定） |
| 2 | `ogImage`（手動指定） |
| 3 | `thumbnailBrand`（例: `prime-video`） |
| 4 | `thumbnailAsins`（2件以上でモザイク、最大4枚） |
| 5 | `thumbnailAsin`（1件のみのとき） |
| 6 | `products` 先頭の ASIN → Creators API 画像（未設定時は Amazon 画像URL） |

`products` に `referencePrice`（参考価格・定価）と `price`（セール価格）を書くと、サムネイルに**赤い打ち消し線付きの元値**とセール価格が表示される。`referencePrice` 省略時は `note` から自動抽出する。

**配信・特集記事**（Prime Video など）で商品カードを出したくない場合は、`thumbnailBrand: prime-video` を指定する（ASIN不要）。一覧・トップに「Prime Video」ブランドの装飾サムネが表示される。左上バッジは `thumbnailBrandBadge`（例: `TBSドラマ`）で任意指定。従来の `thumbnailAsins`（Blu-rayジャケットのモザイク）も引き続き使える。

手動で上書きしたい場合のみ `heroImage` / `ogImage` を指定する。

### 商品カードの自動表示

`products` を frontmatter に書くと、**目次の直上**にカードが出る（見出しなし）。

| 位置 | 見出し |
|------|--------|
| 目次の直上 | なし（`showHeading={false}`） |

記事末尾に商品カードや「購入判断は…」の注意書きは**入れない**（レイアウト側でも末尾カードは出さない）。

### 本文への差し込み

「チェックすべきポイント」冒頭など、本文中に1箇所入れる程度で足りる。

```markdown
::product{B0DL9MHFJJ}::
```

ASIN を省略すると、frontmatter の先頭 `products` が使われる。

### リンク・アソシエイト

- **amazon.co.jp へのリンクはすべて自動で `tag=be-prime-22` が付く**（手動で tag を書く必要はない）
- **Keepa への言及は外部（keepa.com）ではなく** [Keepaの使い方](/posts/keepa-amazon-price-tracker-guide/) **への内部リンク**にする（Keepa解説記事本体は公式URLをそのまま使う）
- 本文中の `Amazonプライム` / `Prime Day` / `タイムセール` は**各キーワードの初出のみ**自動リンク（rehype）
- `cta: prime-day` などで記事末尾 CTA を切り替え

## 8. 品質チェックリスト

- [ ] タイトルにメインKW・価格/割引（該当する場合）が自然に含まれている
- [ ] description と初段が役割分担されている（同文のコピペになっていない）
- [ ] 初段で結論が述べられ、「結論から言うと」がない
- [ ] 価格は `○○円` 表記で統一されている
- [ ] 価格の注意書きは記事全体で1回（まとめ等）。「いまの価格と通常価格の差」セクションは入れない
- [ ] H3 に番号が付いていない
- [ ] 不要な「買い方3ステップ」や水増しの締め文がない
- [ ] `::product{}::` は本文中に1箇所程度。まとめ直後や記事末尾に置かない
- [ ] `category` が正しく設定されている
- [ ] セール記事なら `saleEndDate` を検討した
- [ ] `draft: false` で公開
- [ ] `npm run check:article` で太字パースを確認した（未パースの `**` が残っていない）
